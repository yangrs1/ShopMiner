# ADR-001: 选择 Flask 3.0 RESTful 架构

## Status

Historical — 项目立项时已确定，未发生变更。

## Context

ShopMiner 需要构建一个同时承载电商业务和数据挖掘可视化的后台服务。在选择 Web 框架时，团队评估了三个主流 Python 框架：

- **Flask 3.0**: 轻量微框架，通过扩展生态实现 ORM、认证、跨域等功能。社区成熟，文档丰富，适合中小型项目。
- **FastAPI**: 异步原生，自动生成 OpenAPI 文档，类型校验通过 Pydantic 内建。性能数据亮眼，但当时团队成员对其生产环境的成熟度有顾虑。
- **Django REST Framework**: 全家桶方案，ORM、Admin、认证开箱即用。但重度耦合的架构在数据挖掘场景下不够灵活，且迁移成本高。

项目的关键需求包括：

1. 需要 SQLAlchemy ORM 灵活操作 PostgreSQL，支持 RFM 分析等复杂查询。
2. API 需支持 JWT 认证和 RBAC 权限控制。
3. 前端使用 Vue 3 SPA，后端只需提供纯 REST API，无需服务端模板渲染。
4. 团队对 Flask 的 Blueprint 模块化有既有经验，迭代速度优先。

## Decision

选用 **Flask 3.0** 作为 Web 框架，配套构建以下架构：

- **Flask-SQLAlchemy**: ORM 层，声明式模型定义，支持原生 SQL 回退。
- **Flask-JWT-Extended**: JWT 令牌签发与验证，支持 access/refresh 双令牌。
- **Flask-CORS**: 跨域资源共享，支持 Vue 开发服务器（localhost:5173）与生产环境分离部署。
- **Flask-Limiter**: API 速率限制，防止爬虫与滥用。
- **Flask-Bcrypt**: 密码哈希。
- **Flask-Migrate (Alembic)**: 数据库迁移版本管理。
- **Blueprint 工厂模式**: 按业务域拆分 Blueprint（auth、product、order、cart、analytics、admin、upload），通过 `create_app()` 工厂函数统一注册。

架构示意：

```
create_app()
  ├── 注册扩展 (db, migrate, jwt, bcrypt, cors, limiter)
  ├── 注册 Blueprints
  │   ├── api/v1/auth    → /api/v1/auth/*
  │   ├── api/v1/product → /api/v1/product/*
  │   ├── api/v1/order   → /api/v1/order/*
  │   ├── api/v1/cart    → /api/v1/cart/*
  │   ├── api/v1/analytics → /api/v1/analytics/*
  │   ├── api/v1/admin   → /api/v1/admin/*
  │   └── api/v1/upload  → /api/v1/upload/*
  └── 静态文件路由 (Vue SPA fallback)
```

API 统一响应格式：

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

## Consequences

**正面**:

- 明确的请求-响应循环，配合 `before_request` 钩子实现统一的 Content-Type 校验。
- SQLAlchemy ORM 提供高度灵活的查询能力，RFM 分群、销量聚合等复杂分析查询可流畅编写。
- Blueprint 工厂模式使新增 API 模块只需新建文件并注册，不影响现有代码。
- 扩展生态成熟，每个能力点到点集成，无过度框架绑定。

**负面**:

- 无异步原生支持。`subprocess.Popen` 方式触发 ML 重算时，管理后台会短暂卡顿（直接促成 ADR-003）。
- 无内建 API 文档生成，需额外集成 Flask-RESTX 或手动维护。
- 类型校验需手工编写 Marshmallow 或依赖 Pydantic，相比 FastAPI 增加了样板代码。

**权衡**:

选择 Flask 本质上是选择了"灵活但需自行组装"的路径。对于 ShopMiner 这种包含独特数据挖掘流程的项目，框架不预设范式反而是优势。如果未来性能瓶颈突出，可考虑在关键路径引入 Quart（Flask 的异步姐妹项目）或拆出独立 ML 微服务。

## Compliance

验证方式与示例：

1. **应用工厂加载**: `app/__init__.py` 中存在 `create_app()` 函数，调用 `db.init_app(app)` 等扩展初始化逻辑。
2. **Blueprint 注册**: `app/api/__init__.py` 中存在 `register_blueprints(app)` 函数，遍历 v1 下所有 Blueprint 模块并注册。
3. **统一响应格式**: 所有 API 端点返回包含 `code`、`message`、`data` 三个字段的 JSON 对象。集成测试中可通过解析响应体断言字段存在。
4. **扩展加载**: `app/extensions.py` 中声明 `db = SQLAlchemy()`、`jwt = JWTManager()` 等全局扩展实例。
5. **无服务端模板**: 项目中不存在 `render_template` 调用，所有页面由 Vue SPA 通过 `index.html` 入口加载。
